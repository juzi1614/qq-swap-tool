# QQ 聊天记录视角互换工具 v3.1

把 A 和 B 的历史聊天记录"变成"C 和 B 的聊天记录——消息内容、时间戳完全保留，只看谁是"我"谁是"对方"。

## 适用场景

| 场景 | 说明 |
|------|------|
| 换号保留聊天记录 | 旧号 A 和新号 C 都和 B 聊过天。把 A↔B 的记录嫁接到 C↔B，在新号上继续看旧聊天 |
| 视角取证 | A 和 B 的聊天记录，部署到 C 号上让 B 当"我"、A 当"对方"，从 B 的视角重新阅读 |
| 合并聊天 | 多个号都和同一个人聊过，合并到一个号上集中查看 |

## 核心前提

> **输出账号和目标 peer 之间不能有过聊天记录**（本地和服务器都不能有），否则服务器只下发最近几条消息，其余不显示。

创建输出账号时，对话列表里该 peer 的聊天窗口必须是**完全空白**的。建议使用专门注册的新号作为输出账号。

## 安装

### 方式一：exe 免安装

下载 [Releases](../../releases) 中的 `QQ-Swap-Tool.exe`，双击运行。

### 方式二：源码运行

```bash
pip install sqlcipher3
python gui.py
```

## 使用步骤

### 1. 准备工作

- 关闭 QQ
- 确认输出账号已登录过 QQ（需要存在本地数据库文件）

### 2. 获取密钥

打开 GUI → 点击源账号和输出账号的 **"获取"** 按钮 → QQ 登录窗口弹出 → 分别登录对应账号 → 密钥自动填入。

也可以手动在 PowerShell 中运行：
```powershell
powershell -ExecutionPolicy Bypass -File "windows_ntqq_get_key.ps1"
```

### 3. 填写参数

| 参数 | 说明 | 示例 |
|------|------|------|
| 源账号 | 谁提供聊天记录 | `1686698281` |
| 目标对话 | 和谁的对话要互换 | `3293352433` |
| 输出账号 | 部署到哪个号 | `3533368708` |
| 显示区域 | 聊天出现在哪个对话区域 | `3293352433`（同目标对话） |
| 右侧显示 | 谁当"我"（显示在右侧） | `1686698281` |

### 4. 执行互换

点击 **"开始互换"** → 等待完成 → 打开 QQ 登录输出账号 → 查看结果。

### 5. 恢复（出问题时）

点击 **"恢复数据库"** 从自动备份还原，或手动将 `nt_msg.db.backup_*` 改名为 `nt_msg.db`。

## 常见问题

**Q: 输出账号只有几条消息，其余不显示？**  
A: 输出账号和目标 peer 有服务器聊天记录。换一个没聊过天的新号做输出。

**Q: 密钥获取失败？**  
A: 确保 QQ 完全关闭后再点"获取"；登录后在 QQ 主界面停留几秒。

**Q: 源账号和目标对话填什么关系？**  
A: 源账号和目标对话有过聊天记录。例如：1686698281（源）和 3293352433（目标对话）是好友，他们的聊天记录会被提取并互换视角。
